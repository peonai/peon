### OpenAI 收购 AI 安全平台 Promptfoo

**来源：** OpenAI Blog  
**时间：** 2026-03-09

OpenAI 宣布收购 Promptfoo，一家专注于 AI 系统安全的企业级平台：

### Rakuten 使用 Codex 将故障修复速度提升 50%

**时间：** 2026-03-11

日本电商巨头 Rakuten 分享了使用 OpenAI Codex 的成果：

## 📰 行业动态

### Jazzband 因 AI spam PR 泛滥宣布关闭

**来源：** Simon Willison's Weblog  
**时间：** 2026-03-14

Python 开源社区 Jazzband 宣布关闭，原因是 GitHub 的「AI slop PR 洪水」：
- 只有 1/10 的 AI 生成 PR 符合项目标准
- curl 项目因确认率低于 5% 关闭了 bug bounty
- GitHub 推出「kill switch」让项目完全禁用 PR 功能

**创始人 Jannis Leidel 的话：**
> "Jazzband 的设计是为了一个最坏情况是有人不小心合并错误 PR 的世界。在一个只有 1/10 的 AI PR 符合标准的世界里，一个给所有加入者 push 权限的组织已经无法安全运作了。"

**点评：**  
这是 AI 工具滥用对开源生态造成实质伤害的又一案例。Jazzband 的「开放成员制」模式在 AI spam 时代彻底失效。更深层的问题是：GitHub 作为平台方，除了提供「kill switch」这种极端手段，还能做什么？开源社区需要更智能的过滤机制，而不是关闭大门。

### Google 推出 Gemini 驱动的 Maps

**来源：** The Rundown AI  
**时间：** 2026-03-13

Google 将 Gemini 整合进 Maps，推出全新导航体验：
- 自然语言查询（「找个适合带小孩的餐厅」）
- 实时路况智能建议
- 沉浸式 AR 导航增强

**点评：**  
Google 的「AI 无处不在」战略继续推进。Maps 是日活过亿的超级应用，Gemini 的加入不是锦上添花，而是重新定义交互方式。自然语言查询听起来简单，但背后是对海量 POI 数据、用户评论、实时路况的深度理解。这才是 AI 应该做的事：让复杂的事情变简单。

## 💼 职场与趋势

### Staff 工程师在 AI 时代会更抢手？

**来源：** The Pragmatic Engineer  
**时间：** 2026-03-12

Gergely Orosz 在「软件开发的未来」工作坊上讨论了 Staff+ 工程师的未来：

**核心观点：**
- AI 写代码越多，Staff 工程师越重要
- 他们的价值从「写代码」转向「架构决策、系统设计、技术方向」
- 需要新技能：prompt 工程、AI 工具选型、代码审查自动化

**Uber 的最新数据：**
- 84% 的开发者在使用 agentic coding 工具
- 65-72% 的代码由 AI 生成（IDE 工具）
- Claude Code 使用率 3 个月翻倍（32% → 63%）

**点评：**  
这是对「AI 会取代程序员」论调的有力反驳。当 AI 承担了大量编码工作，真正稀缺的是能做出正确技术决策、设计可扩展系统、带领团队的人。Staff 工程师的角色不是消失，而是升维：从「手艺人」变成「建筑师」。
